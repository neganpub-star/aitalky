## 一、读文档后发现的"漏表 / 需修正"

### 1. 计费(billing)——之前严重过简,需重构
文档《服务订阅》+ 截图证实:计费 = **套餐(含多资源配额) + 资源用量 + 加量包 + 订单(加密在线支付)**。
- `pf_plan` 套餐:档位(体验/基础/标准/专业/旗舰/定制)、月价、**起订≥6月**、各资源配额(席位/翻译字符/客户数/文章/站点/AItokens,部分"无限")、功能项、是否私有化。
- **`pf_addon_pack` 加量包定义(新增)**:翻译包/席位/客户配额包,规格+价格。
- `pay_subscription` 订阅:项目级、当前套餐、状态、起止(**开通次日起算**)、席位。
- **`pay_quota` 资源用量(新增)**:项目×资源(total/used);席位/文章等 used 是**动态算**(席位=成员数)。
- `pay_order` 订单:**加 加密支付字段**(coin=USDT、network=TRC20/ERC20、pay_address、tx_hash、order_no、order_type=套餐订阅/续费/升级/购席位/购加量包、period_days、amount、status=待支付/已支付/已取消/超时、expire_time 24h)。
- 规则:升级(补差价/重设时间两种公式)、续费(≥6月)、到期只能订"资源≥已用"的套餐、体验版资格限制。→ 这些是**业务逻辑**,表只需支撑。

### 2. 信使多语言内容——漏表
《信使设置》《多语言》:问候语、团队介绍、**紧急通知**都是**按语言分别设置**。
- **`mse_messenger_i18n` 新增**:messenger_id + language + greeting(问候语) + team_intro(团队介绍) + urgent_notice(紧急通知) + urgent_enabled。
- `mse_messenger` 补字段:logo、brand_name(品牌名)、default_language、enabled_languages(JSON,项目启用的语种,默认含简中+英文)、reply_time(团队回复时间预期)、message_retention_days、popup_enabled(弹窗)、popup_allow_close(是否允许客户关弹窗)。(红点/声效恒开,无需字段)

### 3. 客户(customer)——身份与来源要补
《用户对话方案》《开始处理会话》《黑名单》:
- 客户身份两种:**userId(业务UID)** 或 **专属会话模式下的 visitor_id(设备/缓存标识,匿名游客)** → `cus_customer` 加 `visitor_id`。
- **名称/头像随机生成**(系统生成,非真实)。
- **来源/设备**:APP端(APP名/版本/机型/系统)、PC端(页面URL/浏览器/系统)→ 加 `source`/`device_info`(或挂在会话上)。

### 4. 会话(conversation)——状态与配置要补
《开始处理会话》:
- 状态除 进行中/已结束,还有 **等待队列(满载排队)**;`assignee` 空=未分配 → 视图 我的/未分配/全部/提及我的。
- **保持期自动结束**:会话达"保持期"自动结束 → 需配置项(挂 `asn_config` 或 messenger)。
- **客户源语言 + 自动翻译开关**:可挂会话(per-conversation)。
- 重新打开:客户在已结束会话回复 → 自动重开 + 重新参与分配。

### 5. 内部备注/@提及——建议改设计
《开始处理会话》:"内部回复"是**消息区里的一种回复模式**(客户不可见),@提及发生在内部回复里,驱动"提及我的"视图。
- **建议:取消 `cnv_note` 单表,改为 Mongo 消息带 `internal=true` + `mentions:[memberId]`**(与普通消息同库同流,更自然)。"提及我的"视图靠 mentions 索引。

### 6. 角色权限(role)——权限是两类
《角色权限》:权限 = **页面权限 + 功能权限**(不是单层模块);3 预置角色(负责人/管理员/普通用户,名+权限不可改);收件箱"我的/提及我的"默认可见、"全部/未分配"可配。
- `id_role.permissions` 改为结构化:`{"pages":[...], "functions":[...]}`;预置 3 角色初始化。

### 7. 邀请(invite)——邮箱 vs 链接
《邀请队友》:邮箱邀请(指定角色) + **链接邀请(可复用、记加入人数、可禁用)**;链接 **72h** 有效;邀请记录(撤销/再次)。
- `id_invite`(邮箱邀请) + **`id_invite_link` 新增**(可复用链接:token/role_id/status/join_count/disabled)。

### 8. 黑名单(blacklist)——按 用户 / 设备
《黑名单》:游客拉黑**设备(visitor_id)**;用户拉黑**业务UID(全设备生效)**。
- `sup_blacklist` 改:`target_type`(1用户 2设备) + `target_value`(external_user_id 或 visitor_id)。

### 9. 项目(project)——站点/私有化
《创建项目》:云上(**中国站/国际站**)+ 专有云(私有化=定制版)。
- `id_project` 加 `site`(站点) + `is_private`(是否专有云)。

### 10. 其他
- **API管理**:安装信使需 APP_ID(=project.uuid)+ 可能 app_secret(SDK)。→ 评估给 project 加 `app_secret`。
- **快捷回复分类**:有"创建分类",可单独 `sup_quick_reply_category` 或简单字符串(V1 倾向字符串)。
- **翻译**:译文缓存在消息(Mongo `translations`)✓;用户消息首次翻译扣字符、再翻不扣,坐席消息每次扣 → 计费逻辑。

### 11. 协议/法律文档(后管可编辑)——漏表
注册页"即同意《服务条款》《隐私政策》"、订阅弹窗"已阅读《套餐订阅服务协议》"等,这些文档需**平台后管可编辑**。
- **`pf_agreement` 新增**(平台级):`type`(服务条款/隐私政策/套餐订阅服务协议/…) + `language`(多语言) + `title` + `content`(富文本) + `version` + `status`(发布)。运营方维护,全平台展示(注册/订阅/信使端等)。

## 二、需要你确认的问题(确认后我重写 DDL)
