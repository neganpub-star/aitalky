# 配置说明(profile 与环境变量)

## profile 方案
- **本地开发**:不激活任何 profile,直接用各 launcher 的 `application.yml`(你的本机明文配置,未改动)。
- **测试环境**:激活 `test`,加载 `application.yml` + `application-test.yml`(后者覆盖环境相关项)。

```bash
# 测试环境启动(三个进程同理)
java -jar aitalky-app.jar   --spring.profiles.active=test
java -jar aitalky-admin.jar --spring.profiles.active=test
java -jar aitalky-ws.jar    --spring.profiles.active=test
# 或用环境变量:export SPRING_PROFILES_ACTIVE=test
```

取值优先级:**环境变量 > application-test.yml 的 `${VAR:默认}` > application.yml**。
`application-test.yml` 只写"环境相关"项,其余配置(mybatis、flyway、multipart、job cron、日志等)继承 `application.yml`。

## 测试环境需注入的环境变量(未设则用 -test 内默认/回退)
| 变量 | 用途 | 备注 |
|---|---|---|
| `AITALKY_DB_URL` / `AITALKY_DB_USER` / `AITALKY_DB_PASSWORD` | MySQL | password 默认空,务必设 |
| `AITALKY_REDIS_HOST` / `_PORT` / `_DB` / `_PASSWORD` | Redis | app/admin/ws 共用 |
| `AITALKY_MONGO_URI` | MongoDB(仅 app) | |
| `AITALKY_ROCKETMQ_NAMESERVER` | RocketMQ(app/ws) | |
| `AITALKY_JWT_SECRET` | JWT 签名密钥 | **三进程必须一致**,默认空=务必设 |
| `AITALKY_RSA_PUBLIC_KEY` / `AITALKY_RSA_PRIVATE_KEY` | 密码传输加解密 | app/admin 需一致 |
| `AITALKY_DC_ID` / `AITALKY_ADMIN_DC_ID` / `AITALKY_WS_DC_ID` | 雪花 datacenterId | 默认 0/1/2,多实例需各唯一 |
| `AITALKY_WS_INSTANCE_ID` | ws 实例标识 | 多 ws 实例需唯一 |
| `AITALKY_MAIL_HOST` / `_PORT` / `_USER` / `_PASSWORD` | SMTP(app) | 验证码邮件 |
| `AITALKY_MINIO_ENDPOINT` / `_ACCESS_KEY` / `_SECRET_KEY` / `_BUCKET` | 对象存储(app) | |
| `AITALKY_CONSOLE_BASE_URL` | 邀请落地页基址(app) | 测试域名 |
| `AITALKY_WIKI_ARTICLE_BASE_URL` | wiki 文章对外阅读基址(app) | console HashRouter 需带 `/#/wiki-article` |
| `AITALKY_VERIFY_MASTER_ENABLED` | 万能验证码开关(app) | test 默认 true 便于联调;**正式置 false** |
| `AITALKY_TRANSLATE_GATEWAY_URL` / `AITALKY_TRANSLATE_ENABLED` | 翻译网关(app) | |
| `AITALKY_GEOIP_ENABLED` | IP 归属地(app) | |
| `AITALKY_BILLING_SIGN_KEY` / `AITALKY_BILLING_ADDRESS_KEY` | 计费签名/地址加密 | 默认空=务必设 |
| `AITALKY_COINLY_BASE_URL` / `_PID` / `_API_KEY` / `_CALLBACK_URL` | Coinly 渠道(app) | |

> 前端(console/admin/messenger)走 Vite 构建 + `/api` 代理,环境地址由部署侧 Nginx/网关决定,不在本 profile 范围。
