## A. 产品形态与范围

### A1 部署形态 ✅
**SaaS 多租户**（项目=租户，客户注册即用，一套系统服务所有租户），架构兼容后续「独立部署」。
> 按 ByteTrack：项目天然隔离、信使接入。

### A2 V1 功能范围 ✅
- **平台后管(运营后台,V1 做)**：平台管理员登录、注册用户管理、租户/项目管理、订阅与订单查看、**套餐管理(CRUD)**、基础运营统计。
- **租户·坐席侧**：项目/账号/成员、RBAC(自定义角色+模块权限)、共享收件箱+会话分配(手动/轮流/负载 + 承载量 + 普通/专属组)、会话/消息(实时WS、已读、撤回、富消息)、web 信使接入(URL/JS、userId 聚合、多语言、redirect、品牌化:标题/图标/域名/角标)、**AI 翻译**、快捷回复、黑名单、消息保存期、多维提醒(红点/声效/弹窗)。
- **不做(V2)**：知识库(wiki)、数字员工(Finn)、智能流程、主动营销/客户洞察、AI 自动回复、通知中心。

### A3 计费/订阅 ✅（按参考 + 复用现有加密支付）
按 ByteTrack：「按项目订阅才可用」。V1 **做完整订阅**：套餐 + 席位数 + 按项目生效 + 到期控制（未订阅/过期则限制使用）。
**支付：复用现有系统的数字货币(加密货币)支付**——cicada 现有 `NexaPayUtil`、获取充值地址、币种列表、套餐购买/续费、席位续费、交易记录。aitalky 订阅流程对接**同一套加密支付**（充值地址/到账回调/订单）。
> 设计阶段会把 cicada 的支付对接方式摸清，作为 aitalky 订阅模块的支付适配。

## B. 规模与性能

### B1 规模量级 〔假设·可改〕
按通用客服规模设计：**万级租户/坐席以内，单体起步，架构按可水平扩展**（无状态服务 + Redis 路由 WS）。ByteTrack 宣称高并发推送，故 WS/消息层留扩展余地。
> 若你们实际量级很大，告诉我，会调存储/MQ 选型。

### B2 多语言 ✅
框架支持任意语言，**内置 ByteTrack 同款 16+ 语言**；**实时翻译**贯穿。

---

## C. 复用与集成

### C1 翻译 ✅
实现**实时消息翻译**（按 ByteTrack）。Provider **沿用 cicada 现有翻译能力/接口**（不重造）。
> 具体 provider（Google/DeepL/自研）按现有的来；接口在 AI/翻译模块内封装。

### C2 基建 〔假设·可改〕
新系统**独立一套基建**（Redis / 对象存储 MinIO / 数据库 / MQ 各自独立），与旧 cicada **解耦**，避免互相影响。
> 若要复用现有 Redis/MinIO 实例，告诉我即可改。

### C3 认证 ✅（按参考）
**邮箱注册 + 密码 + 邮箱验证码登录(2FA)**，与 ByteTrack 一致。

### C4 业务对接（客户身份）✅（按参考）
信使 URL 携带业务系统 `userId`，**按 userId 聚合该用户全部会话**；要求 userId **复杂无序防猜测**（账号-注册时间 等形式）。无 userId 时按匿名访客(uuid)处理。

## D. Web3 业务特性

### D1 会话资料字段 ✅（按参考 + 扩展）
标准字段(会话ID/IP/地区/会话分配/业务UID/联系方式/邮箱) **+ 预留「自定义客户属性」**（钱包地址/链/交易号 等 Web3 字段，通过 URL 参数或 API 传入，V1 支持可配置展示，不写死）。

### D2 接入渠道 ✅（按参考）
**只做 web 信使**（URL + JS 网页插件接入；SDK 后置）。渠道做成**抽象接口**，将来加渠道是插件。甩掉 cicada 的 TG/FB/TikTok 等。

## E. 技术选型（按推荐；ByteTrack 内部栈未知，不强求一致）

| 项 | 决定 |
|---|---|
| E1 后端 | **Java 21 + Spring Boot 3.x，去 RuoYi**，模块化单体 + DDD ✅ |
| E2 存储 | 暂定 **MySQL(业务) + MongoDB(消息) + Redis(缓存/会话/WS路由)**（团队熟 + 消息量大；也可改 PostgreSQL 一库）〔可改〕 |
| E3 MQ | **单套 RabbitMQ**（去掉双 MQ）✅ |
| E4 实时 | **Netty WebSocket**（cicada 已验证）✅ |
| E5 前端 | **Vue3 + TS + Vite + Pinia**，三端统一；UI 库 **Element Plus**〔可改 Naive〕 |
| E6 AI 层 | V1 翻译留 Java 一个模块；**预留「智能」边界**，V2 接知识库/Finn（届时可独立 Python 服务）✅ |
| 文件 | 对象存储 MinIO |
